# SPEC.md — 共享心愿相册

**项目名称**：共享心愿相册（Shared Wish Album）
**版本**：v1.0
**作者**：独立完成
**日期**：2026-07-08

---

## 一、问题陈述

### 1.1 要解决的问题

亲密关系（情侣、挚友、家人）之间缺少一个**轻量、专注**的共享空间来记录共同的回忆与未来的期待。市面上的社交软件要么过于嘈杂（朋友圈），要么功能过于复杂（如 Notion）。用户需要一个**只有两个人**的私密空间：上传一起去过的地方的照片、记录未来想去的地方和想吃的美食，达成后还能关联照片作为纪念。

### 1.2 目标用户

- 情侣：计划约会、记录旅行、分享美食打卡
- 好友：计划旅行、记录聚会、分享探店
- 家人：记录家庭活动、分享生活点滴

### 1.3 为什么值得做

- 微信小程序天然适合"亲密关系"场景——微信是熟人社交，分享流程零摩擦
- 30 秒能说清：**"你和 TA 的专属相册 + 心愿清单，记录去过的地方，写下想去的地方，一起完成彼此的愿望"**
- 市场上有共享相册（如 QQ 空间的共享相册）和心愿工具（如心愿清单 App），但将两者结合在一个亲密共享空间的产品不多

## 三、功能规约

### 模块 1：用户系统

| 功能 | 描述 |
|------|------|
| 登录 | 调用 `wx.login` 获取 code，云函数换取 openid，自动注册/登录 |
| 用户信息 | 获取微信昵称和头像（`wx.getUserProfile` 或 `open-data` 组件） |

**边界条件**：首次登录无网络时给出提示；登录失败重试 3 次后提示用户稍后再试。

### 模块 2：空间管理

**输入**：空间名称
**行为**：
1. 用户点击「新建空间」，输入名称
2. 系统生成唯一 6 位邀请码，创建 space 记录，创建者自动加入成员列表
3. 用户点击「加入空间」，输入邀请码
4. 系统校验邀请码有效性，将用户加入成员列表
5. 空间列表展示所有已加入的空间

**输出**：空间列表页、邀请码展示
**边界条件**：邀请码不存在时提示"邀请码无效"；用户已在该空间时提示"你已加入"；空间名称不能为空；邀请码 6 位字母数字组合，全局唯一

### 模块 3：共享相册

**上传照片**
- **输入**：图片文件 + 拍摄时间 + 地点名称 + 经纬度（可选）+ 感受文字
- **行为**：图片压缩后上传至云存储 `spaces/{spaceId}/photos/{fileId}`，元数据写入 photos 集合
- **输出**：上传成功提示，刷新照片流
- **边界条件**：图片超过 10MB 时提示压缩；网络中断时提示重试；地点和感受字段可选

**照片流**
- **输入**：空间 ID
- **行为**：按 `createdAt` 倒序分页查询，每页 20 条；支持下拉刷新、上拉加载更多
- **输出**：照片列表（缩略图 + 地点 + 简短描述）
- **边界条件**：空相册时显示"还没有照片，快来上传第一张吧"；加载失败显示重试按钮

**照片详情**
- **输入**：照片 ID
- **行为**：查询照片完整信息，展示大图、拍摄时间、地点、感受、上传者
- **输出**：照片详情页
- **边界条件**：照片不存在/已删除时提示"照片已失效"

### 模块 4：心愿记录

**新建心愿**
- **输入**：标题 + 类型（美食/旅游/其他）+ 地点（可选）+ 备注（可选）
- **行为**：写入 wishes 集合，关联 spaceId
- **输出**：心愿列表页刷新
- **边界条件**：标题不能为空（最大 100 字）；类型必选

**心愿列表**
- **输入**：空间 ID
- **行为**：按类型/状态（进行中/已完成）分类展示
- **输出**：心愿列表（标题 + 类型 + 状态）
- **边界条件**：空列表时显示"还没有心愿，快去创建吧"

**心愿详情 & 完成**
- **输入**：心愿 ID
- **行为**：查看心愿详情；点击「完成」标记 done=true，选择关联照片，记录完成时间和完成者
- **输出**：心愿详情页，关联照片展示
- **边界条件**：已完成的心愿不可重复完成；关联照片需从当前空间相册中选择

### 模块 5：Web 管理后台

**功能**：简单展示页，展示所有空间的基本信息（名称、成员数、照片数、心愿数）
**用途**：满足课程"线上可访问 WebUI"要求；作为产品展示页
**技术**：静态 HTML 页面，通过云函数 HTTP API 读取云数据库数据
**边界条件**：不暴露用户隐私数据；仅展示统计信息，不展示具体照片和心愿内容

## 四、非功能需求

### 4.1 性能
- 小程序首页加载 < 3 秒
- 图片压缩后单张 ≤ 2MB
- 照片流分页加载，每页 20 条
- 云函数冷启动 < 5 秒

### 4.2 安全
- **凭据威胁模型**：本项目不涉及 LLM API Key 管理。用户授权通过微信 OAuth 完成，openid 由云函数安全获取。云数据库使用安全规则控制访问权限。
- **数据隔离**：每个空间的数据仅对成员可见，云数据库安全规则校验 `spaceId` 是否在用户所在空间的 `members` 列表中
- **云存储隔离**：图片文件按空间目录隔离存储
- **注入防护**：云数据库查询使用参数化查询，避免 NoSQL 注入
- **传输安全**：所有通信走 HTTPS

### 4.3 可用性
- 适配主流手机屏幕（375px-414px 宽度）
- 支持微信 iOS 和 Android 客户端
- 错误状态有明确提示和重试机制

### 4.4 可观测性
- 云函数日志记录关键操作（登录、创建空间、上传照片）
- 错误日志包含错误码和上下文信息

### 5.2 数据流

1. **登录流程**：小程序 `wx.login` → 云函数 `login` → 换取 openid → 查找/创建 user 记录 → 返回用户信息
2. **上传照片流程**：用户选择图片 → 本地压缩 → 上传云存储 → 写入 photos 集合 → 刷新照片流
3. **心愿完成流程**：标记完成 → 选择关联照片 → 更新 wishes 集合 → 刷新心愿列表
4. **Web 管理后台**：Vercel 静态页面 → 调用云函数 HTTP API → 读取统计数据 → 渲染展示

### 5.3 外部依赖

| 依赖 | 用途 | 说明 |
|------|------|------|
| 微信云开发 | 后端服务 | 云数据库、云存储、云函数 |
| 微信 OAuth | 用户登录 | 获取 openid |
| Vercel | Web 管理后台部署 | 免费静态托管 |
| 无 LLM API | — | 当前版本不接入 AI 功能 |

## 七、凭据与分发设计

### 7.1 凭据安全

本项目不涉及 LLM API Key 或第三方付费 API Key。用户授权通过微信 OAuth 流程完成：

- 小程序调用 `wx.login` 获取临时 code
- 云函数 `login` 使用 `cloud.getWXContext()` 安全获取 openid，不通过网络传输明文凭据
- openid 存储在云数据库，由云开发安全规则保护

**威胁模型**：唯一敏感信息是微信 openid。openid 由微信平台自动管理，开发者无法直接接触。云函数运行在微信隔离环境中，openid 不会暴露给客户端。

### 7.2 分发

**小程序分发**：
- 通过微信审核后发布，用户通过微信扫码或搜索即可使用
- 开发阶段使用体验版（二维码分享）

**Web 管理后台分发**：
- 部署到 Vercel（免费静态托管）
- 提供公网 URL，满足课程 WebUI 要求
- 部署命令：`vercel --prod`
- 不涉及凭据——仅读取云函数暴露的公开统计数据 API

### 7.3 运行方式

**小程序**：微信开发者工具导入项目 → 配置云开发环境 ID → 编译运行
**Web 管理后台**：`cd web-admin && npx serve .`（本地）或通过 Vercel URL 访问

## 八、技术选型与理由

| 层面 | 选型 | 理由 |
|------|------|------|
| 前端框架 | 微信小程序原生 | 最适合微信生态，云开发 API 原生支持，无需额外框架 |
| 后端 | 微信云开发 | 免费额度足够小体量使用，零运维，微信登录开箱即用 |
| 数据库 | 云数据库（MongoDB 兼容） | 文档型存储适合灵活的数据结构，云开发原生集成 |
| 存储 | 云存储 | 图片存储与 CDN 加速，自带权限控制 |
| 云函数 | Node.js | 云开发默认运行时，社区资源丰富 |
| Web 管理后台 | 纯 HTML/CSS/JS | 极简，无需框架，Vercel 免费托管 |
| 前端设计系统 | 微信小程序原生组件 | 不使用 Open Design——小程序 UI 规范与 Web 差异大，原生组件更符合微信设计规范 |
| LLM 供应商 | 无 | 当前版本不调用 LLM API |

## 十、风险与未决问题

### 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 微信云开发免费额度不够 | 开发/测试受影响 | 监控用量；云函数和数据库查询做优化，减少调用次数 |
| 微信审核不通过 | 小程序无法上线 | 提前查阅审核规范，避免敏感内容 |
| 云函数冷启动慢 | 用户体验差 | 使用云函数常驻实例（付费）或优化启动逻辑 |
| 图片过多导致存储成本上升 | 超出免费额度 | 压缩图片，限制单张 10MB，未来可加清理策略 |

### 未决问题

1. 已完成的心愿是否允许撤销？——待后续确认
2. 是否支持心愿完成时上传新照片（而非仅关联已有照片）？——当前版本仅支持关联已有照片
3. 是否支持照片评论/点赞？——当前版本不包含
4. 是否接入 AI 功能（如智能生成心愿描述）？——当前版本不包含
